# 门店销售报告系统 - 部署检查清单

## 部署前检查

### 系统要求
- [ ] Node.js 版本 18 或更高 (`node --version`)
- [ ] npm 版本 8 或更高 (`npm --version`)
- [ ] 至少 2GB 可用内存
- [ ] 至少 500MB 可用磁盘空间

### 项目文件
- [ ] `package.json` 文件存在
- [ ] `package-lock.json` 文件存在
- [ ] `src/` 目录存在且包含源代码
- [ ] `public/` 目录存在
- [ ] 所有必要的依赖包已列出

### 网络和端口
- [ ] 端口 3000 未被占用（或准备好使用其他端口）
- [ ] 网络连接正常（用于安装依赖）
- [ ] 防火墙允许端口 3000 的流量（如适用）

## 安装和配置

### 1. 安装依赖
- [ ] 运行 `npm install` 成功
- [ ] 所有依赖包安装完成
- [ ] 无安装错误或警告

### 2. 环境配置
- [ ] 如需要，创建 `.env.local` 文件
- [ ] 设置正确的环境变量（端口、环境等）
- [ ] 验证配置文件无敏感信息泄露

### 3. 构建应用
- [ ] 运行 `npm run build` 成功
- [ ] 构建过程中无错误
- [ ] 生成的构建文件完整

## 启动和测试

### 1. 启动应用
- [ ] 开发模式：`npm run dev` 正常启动
- [ ] 生产模式：`npm start` 正常启动
- [ ] 应用在预期端口运行

### 2. 功能测试
- [ ] 访问首页 http://localhost:3000 成功
- [ ] 登录功能正常工作
- [ ] 使用测试账户成功登录
- [ ] 销售报告填报功能正常
- [ ] 历史记录页面正常显示
- [ ] 统计页面正常显示（如适用）

### 3. 用户账户测试
- [ ] 门店员工账户可正常登录
- [ ] 门店经理账户可正常登录
- [ ] 管理员账户可正常登录
- [ ] 权限控制按预期工作

## 性能和安全

### 1. 性能检查
- [ ] 页面加载时间合理（通常小于 3 秒）
- [ ] 表单提交响应迅速
- [ ] 无明显的性能瓶颈

### 2. 安全检查
- [ ] 不在生产环境中使用开发模式
- [ ] 敏感信息未硬编码在源代码中
- [ ] 使用强密码进行登录
- [ ] 如公开部署，考虑使用 HTTPS

## 部署后验证

### 1. 应用健康检查
- [ ] 应用持续运行无崩溃
- [ ] 日志中无错误信息
- [ ] 资源使用在合理范围内

### 2. 数据完整性
- [ ] 提交的销售数据正确保存
- [ ] 历史记录准确显示
- [ ] 统计数据计算正确
