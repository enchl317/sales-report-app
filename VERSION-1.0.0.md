# Sales Reporting App - Version 1.0.0

## 版本信息
- 版本号: v1.0.0
- 发布日期: 2026-03-13
- 版本类型: 首个稳定版

## 功能特性

### 核心功能
1. **用户认证系统**
   - 支持员工和管理员角色
   - 安全的密码验证机制

2. **门店管理**
   - 6家门店支持（南东店、杨浦店、中环店、三鑫店、全土店、汇联店）
   - 门店与员工关联管理

3. **销售数据录入**
   - 日常销售数据提交
   - 支持总销售额、肉松销售额、其他销售额分类
   - 多人同时上报支持

4. **数据可视化**
   - 年度销售汇总页面
   - 月度销售统计
   - 按门店分类查看

5. **月度目标管理**
   - 月度销售目标设置
   - 目标与实际销售额对比
   - 达成率计算和显示

### 技术架构
- **前端**: Next.js 14 (App Router)
- **后端**: Node.js + Next.js API Routes
- **数据库**: MySQL
- **样式**: Tailwind CSS
- **部署**: 支持 Docker 部署

## 数据库结构

### 主要数据表
1. `users` - 用户信息表
2. `stores` - 门店信息表
3. `user_stores` - 用户门店关联表
4. `store_sales_records` - 门店销售记录表
5. `monthly_targets` - 月度目标表

## API 接口

### 主要端点
- `/api/auth/login` - 用户登录
- `/api/stores` - 门店信息
- `/api/sales-report` - 销售报告
- `/api/yearly-sales` - 年度销售数据
- `/api/monthly-targets` - 月度目标管理

## 部署说明

### 环境要求
- Node.js 18+
- MySQL 8.0+
- npm/yarn

### 部署步骤
1. 安装依赖: `npm install`
2. 配置环境变量
3. 初始化数据库: `node scripts/init-db.js`
4. 构建应用: `npm run build`
5. 启动应用: `npm start`

## 更新策略

### 数据库更新
- 新增数据库变更应使用独立的更新脚本
- 不得修改 `init-db.js` 初始化脚本
- 更新脚本命名规则: `update-db-vX.Y.Z.js`

### 代码更新
- 遵循语义化版本控制 (SemVer)
- 重大变更需创建分支开发
- 所有更新需经过测试
